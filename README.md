# data sonification

환경 데이터와 음악(Sound) 에서 시각(Visualization)으로 연결하는 생성형 A/V 프로젝트.
Vanilla JS + [Tone.js](https://tonejs.github.io/) + [p5.js](https://p5js.org/) 으로 구성하여 단일 HTML 코드로 동작합니다.

---

## 개요

두 가지 모드를 지원합니다.

| 모드 | 설명 |
|------|------|
| **데이터** (City's Breath) | 도시 대기 데이터(PM2.5 · 기온 · 습도)를 음악과 시각으로 변환 |
| **음악** (Music's Breath) | 업로드한 음악 파일을 분석하여 Frequency·Beat에 반응하도록 시각화 |

---

## 사용 방법

별도 서버나 빌드 없이 `index.html` 파일을 브라우저로 열면 바로 실행.

### 데이터 모드

1. 우측 상단 토글에서 **데이터** 선택 (기본값)
2. 하단 ▶ 버튼으로 재생 — PM2.5·기온·습도 값이 음과 시각으로 표현
3. `SPEED` 슬라이더로 재생 속도 조절 (0.5× ~ 6×)

`./data/air.csv` 파일이 존재하면 실제 데이터를 로드하고, 없으면 1년치 합성 데이터를 자동 생성합니다.

CSV 형식:
```
date,pm25,temperature,humidity
2024-01-01,35.2,3.1,62.5
...
```

### 음악 모드

1. 우측 상단 토글에서 **음악** 선택
2. 드롭존에 음악 파일을 드래그하거나 클릭하여 업로드
3. ▶ 버튼으로 재생 — 곡이 끝나면 자동으로 정지

지원 형식: MP3 · WAV · MP4 · OGG · FLAC · AAC

## 소니피케이션 매핑 (데이터 모드)

| 데이터 | 음악 파라미터 | 시각 요소 |
|--------|------------|----------|
| PM2.5 | 펜타토닉 음계 음높이 | 입자 밀도 · 오라 반경 |
| 기온 | 음표 길이 (2n / 4n / 8n) | 하늘 색온도 |
| 습도 | 잔향(Reverb) · 딜레이 wet | 파문 투명도 |
| PM2.5 > 70% | 타악(Membrane) + 브라운 노이즈 레이어 추가 | 다중 파문 |

드론(베이스 패드)은 C2·G2·C3 화음으로 배경을 채우고, 멜로디는 AMSynth로 공기 질 변화를 선율로 표현합니다.

## 시각화 (음악 모드)

| 주파수 대역 | 시각 요소 |
|-----------|----------|
| Bass (20–250 Hz) | 중심 입자 폭발 · 동심 파문 · 빛줄기 |
| Mid (250–2000 Hz) | 중심 오라 반경 · 수평 오로라 밴드 |
| Treble (2–8 kHz) | 이중 원형 스펙트럼 · 궤도 스파크 입자 |
| 비트 감지 | 비트 파문 + 위성 파문 3개 동시 발생 |

- **원형 FFT 스펙트럼**: 로그스케일 주파수를 220개 바로 방사형 배치. 내부 링과 외부 미러 링 이중 레이어
- **오로라**: mid 에너지에 반응하는 4개 수평 Perlin noise 밴드
- **시간도메인 파형**: 하단 메인 파형 + 상단 미러 파형 (보색 계열)
- **스파크 입자**: treble 에너지에 비례해 원형 궤도에서 방출

## 기술 스택

| 라이브러리 | 용도 |
|-----------|------|
| [Tone.js 14.8](https://tonejs.github.io/) | 데이터 모드 음원 합성 (AMSynth · MembraneSynth · Reverb · Delay) |
| [p5.js 1.9](https://p5js.org/) | 캔버스 렌더링 · Perlin noise · 입자 시스템 |
| Web Audio API | 음악 파일 디코딩 · FFT 분석 · 비트 감지 |

프레임워크 없이 Vanilla JS ES6+로 작성되었습니다.
